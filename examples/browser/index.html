Browser Harness

这个页面用于浏览器环境下验证 BnbZkIdClient 的加载、配置读取和 prove 主流程。现在支持两种模式：默认的 fixture + fixture 回归模式，以及 Gateway fixture + Primus sdk 的浏览器 live skeleton。

这个 live skeleton 仍然使用 Gateway fixture，只把 Primus 切到真实 SDK，并通过 PADO API 动态解析 template id 和完成服务端签名。`zktlsAppId` 和 PADO API 地址都直接复用 SDK 内置配置，真实初始化仍依赖 Primus 浏览器扩展环境。
