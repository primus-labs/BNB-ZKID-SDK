Browser Harness

这个页面用于浏览器环境下验证 BnbZkIdClient 的加载、配置读取和 prove 主流程：Fixture 为双 fixture； BrevisFixture+PrimusSDK 的 Gateway 使用与 GET /v1/config、POST/GET /v1/proof-requests 对齐的静态 JSON，Primus 仍调 PADO； Brevis+PrimusSDK 为真实 Gateway HTTP 8038 + Primus SDK。

命名约定：应用侧 prove.provingParams 与 Gateway GET /v1/config 的 properties[].businessParams、POST /v1/proof-requests 的 businessParams 为同一载荷（SDK 按配置原样透传，不改写 key/value 类型）。BrevisFixture Gateway 与下拉的配置源均为 examples/browser/fixtures/config.json。开发模式可经 Vite 代理访问 /brevis-gateway/、/pado-api/。仍需 Primus 浏览器扩展等环境。
